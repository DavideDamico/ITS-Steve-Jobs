======================================================================
ORDINE CONSIGLIATO DI IMPORT PER prof_privato
======================================================================

IMPORTANTE: Importa i file CSV in phpMyAdmin nell'ordine seguente
per evitare conflitti con le Foreign Key.

FASE 1 - Tabelle senza Foreign Key:
----------------------------------------------------------------------
1. Student.csv
2. Category.csv
3. Subject.csv

FASE 2 - Tabelle con una Foreign Key:
----------------------------------------------------------------------
4. Payment.csv         (dipende da Student)

FASE 3 - Tabelle con più Foreign Key:
----------------------------------------------------------------------
5. Lesson.csv          (dipende da Student, Subject, Payment, Category)

======================================================================
NOTE IMPORTANTI:
----------------------------------------------------------------------
- Durante l'import, assicurati che 'NULL' venga interpretato come NULL.
- PaymentID in Lesson può essere NULL (~30% lezioni non pagate).
- Le colonne ID sono AUTO_INCREMENT.
- Ogni Payment è ora collegato direttamente allo Student.
