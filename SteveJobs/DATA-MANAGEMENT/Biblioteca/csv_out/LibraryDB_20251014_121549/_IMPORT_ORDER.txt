======================================================================
ORDINE CONSIGLIATO DI IMPORT PER LibraryDB (SCHEMA AGGIORNATO)
======================================================================

IMPORTANTE: Importa i file CSV in phpMyAdmin nell'ordine seguente
per evitare conflitti con le Foreign Key.

FASE 1 - Tabelle senza Foreign Key (indipendenti):
----------------------------------------------------------------------
1. User.csv
2. Category.csv
3. Supplier.csv
4. Author.csv
5. BookInfo.csv        (info base libri - SENZA FK)

FASE 2 - Tabelle con una Foreign Key:
----------------------------------------------------------------------
6. Employee.csv        (dipende da User)
7. Customer.csv        (dipende da User)
8. Review.csv          (dipende da Customer)

FASE 3 - Tabelle con più Foreign Key:
----------------------------------------------------------------------
9. Book.csv            (dipende da BookInfo, Supplier, Author, Review)
10. Rental.csv         (dipende da Employee, Customer)
11. RentalDetail.csv   (dipende da Rental, Book)

======================================================================
SCHEMA AGGIORNATO:
----------------------------------------------------------------------
- BookInfo: contiene le informazioni base dei libri (SENZA FK)
- Book: rappresenta le copie fisiche (CON FK verso BookInfo, 
  Supplier, Author, Review)
- RentalDetail: ha il campo QuantityOfBook e FK verso Book
- Rental: viene importato PRIMA di RentalDetail per evitare
  dipendenze circolari

======================================================================
NOTE:
- Durante l'import, assicurati che il campo 'NULL' venga interpretato
  come valore NULL nel database.
- Se usi phpMyAdmin, seleziona 'NULL' come valore per i campi vuoti
  nelle opzioni di importazione.
- Le colonne ID sono AUTO_INCREMENT, quindi non dovresti avere problemi.
- ReviewID in Book può essere NULL (circa 60% dei libri non hanno recensione).
======================================================================
